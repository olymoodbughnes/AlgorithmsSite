Counting Sort

Counting sorts are normally used when the input dataset has additional data such as knowing that each elements must go on a separate list. In other words, it belongs to a different set. The cost of this algorithm is O dataset hide/show n in big O notation. The way thos algorithm works is by initialising another list, this list will be where the chosen elements will be placed, for example, all odd numbers in a list. The algorithm will begin looking for instances of the correct type of element, then the elements are ordered to better fit into an ordered list, after which point the algorithm can begin placing them in the new separete list. This algoritm is often ignored as it isn't as versatile as other algorithms, this algorithm needs to operate on a smaller dataset, as its cost will make sorting through larger lists a lengthy process

Bucket Sort

Bucket sorts approach to organising datasets is based on segmenting said dataset into smaller datasets and tackling each smaller dataset one at a time, these smaller datasets are usually referred to as buckets, as they behave in a way that resembles a bucket full of something. In summary the smaller datasets can have data inserted in them easily, but this data is kept in that dataset separate from any other similar datasets that might be near it. They all behave as different datasets. For this reason, bucket sort can be used in conjucntion with other algorithms that might be better at sorting smaller datasets.
