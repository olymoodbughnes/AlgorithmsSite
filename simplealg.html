Insertion sort

Insertion sort works by looking at every element in a list, in other words, it scans the list for targets. Once it's found an element to sort, it places this element where it has been specified it should go. It should be noted how the elements chosen for sorting will be placed on a different list, this list will be created along with the usage of the algoritm. Due to this reliance on checking every element in a list every time, the time it takes this algorithm to complete a task can increase by a considerable amount if the list is bigger. For exaple, if this algorithm takes x seconds to comeplete a sort of 100 elements, then a sort of 1000 will take 10x.

Selection sort

This selection sorting algorithm works by taking the raw information from an input and sorts that. This, of course, affects the complexity of the algorithm, as the algorithm will take a proportional amount of time to sort any given sized list. In big O notation, this would be O n squared This fact makes this sorting algorithm immediately fall behind other sorting algorithms when larger lists are needed to be sorted. The way this algorithm works, is by scanning the entire list once, and finding the smallest member of this list, it then places this member on the first position, once this has been done, the algorithm will then proceed to do the same with the rest of the list until it has finished sorting.
