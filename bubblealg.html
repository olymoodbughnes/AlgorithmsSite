Bubble sort

Bubble sorts rely on having to scan the first element of an input list. If the algorithm finds that one element compared to a second element comes first on the list, it will put this element at the appropriate position, since lists usually have non relating sequences of elements that can be arranged at different cardinal positions, the algorithm will, for the most part, find an element of the two compared elements that covers first. This process is reiterated as many times as the code needs to be reiterated for the list to be fully sorted. The cost of this algorithm in big O notation is O n squared. Due to this type of cost, t his algorithm is often used for smaller datasets as it is more efficient and less resource intensive with said datasets.

Comb sort

This sorting algorithm is derived from bubble sort, the way this sorting algorithm works is by eliminating certain values at the ends of the dataset, the reason for this is because in a bubble sort, not eliminating these elements actually slows the algorithm down more than necessary, this sorting algorithm chooses two elements, and, if these two elements are close enough to each other, then the difference between these two elements is then "locked" and more elements are checked/scanned, the algorithm will do this until it has reached a point at which it will sort the dataset exactly the same way it is sorted via a bubble sort.
